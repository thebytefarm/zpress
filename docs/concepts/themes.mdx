# Themes

How the visual identity of your documentation site is controlled through built-in themes, color modes, and token overrides.

## Overview

A theme in zpress is a complete set of color tokens for both light and dark modes. zpress ships with three built-in themes — Base, Midnight, and Arcade — each with its own default color mode. You can override individual color tokens without creating a custom theme, and optionally enable a runtime theme switcher for your readers.

## Built-in Themes

### Base

The default theme. A clean purple/violet palette with full dark/light mode toggle support.

**Default color mode:** `toggle` (users can switch between light and dark)

<BrowserWindow url="https://docs.example.com">
  ![Base theme](../.assets/theme-base.png)
</BrowserWindow>

### Midnight

A super-dark, high-contrast theme with blue-tinted brand colors. Both light and dark variants use very dark backgrounds — light mode is "less dark" and dark mode is near-black.

**Default color mode:** `dark`

<BrowserWindow url="https://docs.example.com">
  ![Midnight theme](../.assets/theme-midnight.png)
</BrowserWindow>

### Arcade

A retro neon theme inspired by arcade cabinets and CRT monitors. Vibrant neon green brand color with deep indigo backgrounds. Includes custom hover animations: border tracing on cards, CRT scanlines on code blocks, neon pulse on buttons, and glow effects on sidebar items.

**Default color mode:** `dark`

<BrowserWindow url="https://docs.example.com">
  ![Arcade theme](../.assets/theme-arcade.png)
</BrowserWindow>

## Configuration

Set the theme in your `zpress.config.ts`:

```ts
import { defineConfig } from '@zpress/kit'

export default defineConfig({
  title: 'My Docs',
  theme: {
    name: 'arcade',
  },
  sections: [
    /* ... */
  ],
})
```

The `name` accepts any built-in theme (`base`, `midnight`, `arcade`) or any custom theme registered in the top-level `themes` array. An unknown name writes a build-time warning to stderr and falls back to `base`.

### Options

| Property     | Type                               | Default           | Description                                  |
| ------------ | ---------------------------------- | ----------------- | -------------------------------------------- |
| `name`       | `'base' \| 'midnight' \| 'arcade'` | `'base'`          | Built-in theme to use                        |
| `colorMode`  | `'dark' \| 'light' \| 'toggle'`    | Per-theme default | Force a color mode or allow toggling         |
| `switcher`   | `boolean`                          | `false`           | Show a theme switcher dropdown in the navbar |
| `colors`     | `ThemeColors`                      | —                 | Override base/light mode colors              |
| `darkColors` | `ThemeColors`                      | —                 | Override dark mode colors                    |

### Color mode

Each theme has a default color mode. You can override it:

```ts
theme: {
  name: 'base',
  colorMode: 'dark',
}
```

| Value    | Behavior                                     |
| -------- | -------------------------------------------- |
| `toggle` | Users can switch between light and dark mode |
| `dark`   | Dark mode is forced, toggle is hidden        |
| `light`  | Light mode is forced, toggle is hidden       |

### Theme switcher

Enable the theme switcher to let users switch between all three built-in themes at runtime:

```ts
theme: {
  switcher: true,
}
```

The switcher appears as a paint bucket icon in the navigation bar. Theme selection persists across page loads via `localStorage`.

## Color Overrides

Override individual color tokens without creating a full custom theme. Overrides are applied as inline CSS custom properties and take precedence over the theme defaults.

```ts
theme: {
  name: 'midnight',
  colors: {
    brand: '#ff6b6b',
    brandDark: '#cc5555',
  },
  darkColors: {
    bg: '#000000',
  },
}
```

### Available color tokens

| Token        | Description                       |
| ------------ | --------------------------------- |
| `brand`      | Primary brand color               |
| `brandLight` | Lighter brand variant             |
| `brandDark`  | Darker brand variant              |
| `brandSoft`  | Soft/tinted brand for backgrounds |
| `bg`         | Main background                   |
| `bgAlt`      | Alternate background              |
| `bgElv`      | Elevated surface background       |
| `bgSoft`     | Soft background                   |
| `text1`      | Primary text                      |
| `text2`      | Secondary text                    |
| `text3`      | Tertiary/muted text               |
| `divider`    | Divider lines                     |
| `border`     | Border color                      |
| `homeBg`     | Home page background              |

Color values must be valid CSS hex (`#xxx` or `#xxxxxx`) or `rgba()` values.

## Custom Themes

When the built-in themes plus color overrides aren't enough, register a fully custom theme through `defineTheme`. The theme is validated at config time and emitted as a single `html[data-zp-theme='{name}']` CSS block alongside the built-ins.

```ts
import { defineConfig, defineTheme } from '@zpress/kit'

const sunset = defineTheme({
  name: 'sunset',
  modes: ['dark'],
  defaultMode: 'dark',
  tokens: {
    colors: {
      brand: {
        primary: '#ff7a3d',
        hover: '#ff8c54',
        active: '#e65f24',
        fg: '#2a0f06',
        soft: 'rgba(255, 122, 61, 0.14)',
        onBrand: '#2a0f06',
        light: '#ffa56a',
        lighter: '#ffc89a',
      },
      // ... see `ZpressTokens` for the full token tree
    },
    // ... spacing, radii, fonts, shadows, motion, etc.
  },
})

export default defineConfig({
  themes: [sunset],
  theme: { name: 'sunset' },
  // ...
})
```

### Constraints

| Constraint                     | Detail                                                                                                                                                                                                                 |
| ------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Theme `name` is a slug         | Must match `/^[a-z0-9][a-z0-9-]*$/`. Used directly in the `html[data-zp-theme='{name}']` selector.                                                                                                                     |
| Token tree must be complete    | Every leaf in `ZpressTokens` is required. `defineTheme` validates through `tokensSchema` and throws on missing leaves.                                                                                                 |
| Token values are CSS-safe      | Token strings reject `;`, `{`, `}`, and `</style>` to prevent CSS injection in tenant-supplied themes.                                                                                                                 |
| Persisted themes are validated | When `switcher: true`, the active theme is persisted in `localStorage`. On the next visit the value is intersected against the registry — stale entries are cleared and the user falls back to the build-time default. |

## Comparing Approaches

| Theme    | Best For                          | Color Mode Default |
| -------- | --------------------------------- | ------------------ |
| Base     | General-purpose docs              | `toggle`           |
| Midnight | Developer tools, dark-first sites | `dark`             |
| Arcade   | Playful branding, creative tools  | `dark`             |

## Design Decisions

- **Three opinionated themes, not a theme builder** — providing complete themes rather than a raw token system reduces decision fatigue. Color overrides provide the escape hatch for brand customization.
- **CSS custom properties** — overrides are injected as inline CSS variables, making them debuggable in browser devtools and compatible with any CSS-in-JS approach.
- **Reduced motion support** — the Arcade theme's animations respect `prefers-reduced-motion`, disabling all continuous animations (border tracing, CRT scanlines, neon pulse, glow bar) automatically.
